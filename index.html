Task Overview

As LLM-based AI systems become increasingly integral to scientific workflows, they provide substantial support across multiple stages of research. However, experiment reproduction remains a significant challenge for current AI agents.

This shared task evaluates agents' ability to reconstruct experimental pipelines from research papers by interpreting text, planning implementations, generating code, and executing experiments within a controlled environment. Unlike prior work that focuses only on final outputs, this task emphasizes the full reproduction process through fine-grained action logs and predefined rubrics covering key stages such as paper understanding, planning, coding, and execution.

For each paper in the dataset, both PDF and Markdown formats are provided. Participating systems must read the given paper and reproduce the codebase for experiment reproduction. During this process, the system must use the tools provided by the MCP Action Recorder to log all actions. Participants submit the action log along with the corresponding repository for evaluation against a series of rubrics covering criteria at each stage of the reproduction pipeline.

Dataset

The dataset comprises 150 papers from 7 scientific domains, split into a training / validation set (with ground-truth rubrics) and a held-out test set (rubrics not released). Each paper entry contains a paper.pdf, paper.md, and an images/ folder.

Training & Validation Set: The ground-truth rubrics.json is released alongside each paper in data/train_valid/ so that participants can examine their agent system's results and adjust accordingly.
Test Set: Rubrics for test papers are withheld. Participants submit their reproduced repositories and action logs; scores are computed by the organizers and released after the evaluation deadline.

Evaluation

Each paper is paired with a set of rubric entries in rubrics.json. Each entry specifies a criterion, a score weight, and one of five evaluation types that together cover the full reproduction pipeline. Scores are weighted by the score field; the total score is normalized to 100.

Rubric Entry Format

{
  "criteria": "A description of what the agent is expected to accomplish.",
  "score": 3,
  "type": "Code Implementation",
  "comment": "Section X, Table X, ..." }

Agent Action Types

Type Description
Paper Observation The agent has correctly read and understood the relevant sections of the paper.
Plan Writing The agent has formulated a plan that covers the necessary components for reproduction.
Code Implementation The agent has written code that correctly implements the required functionality.
Command Execution The agent has successfully executed the implemented code without errors.
Result Matching The agent's execution outcomes are consistent with the results reported in the paper.

MCP Action Recorder

All agent actions during a reproduction run must be routed through the MCP Action Recorder — an MCP server that provides agents with real file and shell tools while recording every action into a chronological JSON log. This log is a core part of the evaluation.

read_file

Read a file from the workspace. Supports optional line-range offsets (start_offset, end_offset).

write_file

Write content to a file in the workspace. Parent directories are created automatically if they do not exist.

execute_cmd

Execute a bash command. Runs with cwd set to the working directory. Read/Write operations are restricted to --work-dir.

export_log

Export the action log to log/actions.json. Also exported automatically on server shutdown.

Quick Start

Run the MCP server with uvx (recommended):

uvx --from tools mcp-action-recorder --work-dir /path/to/workspace

Or install manually:

pip install mcp
python tools/record_tools.py --work-dir /path/to/workspace

MCP Configuration for Claude Code

Add to .mcp.json in your workspace root:

{
  "mcpServers": {
    "action-recorder": {
      "command": "uvx",
      "args": ["mcp-action-recorder", "--work-dir", "./"]
    }
  }
}

To ensure all actions are recorded, restrict built-in tools via .claude/settings.local.json:

{
  "permissions": {
    "allow": ["mcp__action-recorder"],
    "deny": ["Read", "Edit", "Write", "Bash"]
  },
  "enabledMcpjsonServers": ["action-recorder"]
}
See the example_repo/ directory on GitHub for a complete reference setup including sample actions.json output.

Submission

Your submission should contain the reproduced repositories for each paper in the test set along with their action logs. The directory name for each repository must exactly match the subdirectory name in data/test/, and the action log must be saved to log/actions.json under that repository.

Package everything as a single zip file named <your_team_id>.zip and send it to hanhua.hong@postgrad.manchester.ac.uk by 21 June 2026. You may submit multiple times; only the last submission will be scored. All the scores will be released together after the evaluation release date.

Submission Format

your_team_id.zip
│
├─ paper_1/
│  ├─ log/
│  │  └─ actions.json
│  └─ <reproduced source files>
│
├─ paper_2/
│  ├─ log/
│  │  └─ actions.json
│  └─ <reproduced source files>
│
└─ ...
Important: Each repository directory name must exactly match the corresponding subdirectory name under data/test/<domain>/. Submissions with mismatched names cannot be evaluated.
